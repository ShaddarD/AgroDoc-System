# /split-services

Задача: аккуратно разделить перегруженный код на services/selectors без слома поведения.

Порядок:
1. Определить текущие обязанности кода
2. Оставить чтение в selectors
3. Оставить бизнес-операции и транзакции в services
4. Оставить HTTP orchestration во views
5. Оставить валидацию в serializers
6. Показать минимальный рефакторинг

Формат ответа:
- Анализ текущей проблемы
- Что куда вынести
- Код после рефакторинга
- Риски
- Что проверить
