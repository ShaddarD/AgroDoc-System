# Project Brief

Система предназначена для оформления заявок на инспекцию, отбор и сертификацию продукции.

## Что проектируется
- архитектура БД PostgreSQL
- справочники
- связи между сущностями
- ER-диаграмма
- REST API концептуального уровня

## Что не входит
- реализация backend
- SQL-миграции
- UI-код

## Базовые решения
- UUID во всех таблицах как технический PK
- `application_number` — уникальный бизнес-ключ, но не PK
- часть полей заполняется вручную, часть выбирается из справочников
- контейнеры, сертификаты и регламенты хранятся в отдельных дочерних таблицах
- доверенности отправителя вынесены в отдельную сущность
