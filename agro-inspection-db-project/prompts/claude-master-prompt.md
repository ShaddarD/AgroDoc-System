# Claude Master Prompt

Открой проект в VS Code и используй его как архитектурную документацию для сборки рабочей структуры проекта. Не меняй бизнес-логику без явного комментария. Сохраняй UUID как технический PK, а `application_number` как уникальный бизнес-ключ. Отдельно учитывай справочники, дочерние таблицы и API-документацию.
