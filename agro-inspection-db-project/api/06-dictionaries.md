# Dictionaries API

## Базовый принцип
Каждый справочник должен поддерживать:
- список
- карточку
- создание
- редактирование
- деактивацию через `is_active`

## Примеры
- `/api/dictionaries/senders`
- `/api/dictionaries/receivers`
- `/api/dictionaries/products`
- `/api/dictionaries/countries`
- `/api/dictionaries/laboratories`
