# Agro Inspection DB Project

Проект документации по архитектуре БД и API для системы оформления заявок на инспекцию/отбор/сертификацию продукции.

## Цель
Подготовить нормализованную структуру PostgreSQL без реализации кода: таблицы, связи, бизнес-правила, ER-диаграмму и концепцию API.

## Ключевые принципы
- PostgreSQL
- UUID как технические первичные ключи
- номер заявки как уникальный бизнес-идентификатор
- разделение ручного ввода и справочников
- множественные сущности вынесены в отдельные таблицы

## Структура
- `docs/` — бизнес-контекст, правила, валидации, workflow
- `database/` — модель БД, связи, ограничения, типы
- `database/tables/` — по одному файлу на таблицу
- `api/` — концепция REST API
- `diagrams/` — ER-диаграмма и потоки
- `prompts/` — промпты для Claude
- `examples/` — примеры структур данных
